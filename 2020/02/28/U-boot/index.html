U-boot分析

嵌入式系统的启动过程

  1. 运行bootloader — BIOS
  2. 加载linux内核 — windows
  3. 挂载根文件系统 — C盘D盘
  4. 运行应用程序 — 应用程序
  5. 事实上u-boot是一个单片机程序

U-boot编译过程

  1. 解压缩
  2. 打补丁  patch -p <补丁文件
  3. 配置  make 100sk24x0_config
  4. 编译  make

启动内核

  • 从FLASH读出内核
  • 放到SDRAM
  • 启动内核
  • 因此u-boot的功能必须有
    1. 读flash+写flash内核(为了开发方便)
      1. 写flash
      2. 网卡
      3. usb
    2. 初始化SDRAM
      1. 初始化时钟
      2. 启动串口
    3. 启动内核
      1. 读出内核从flash
        1. 初始化SDRAM
        2. 初始化时钟
        3. 关看门狗

通过Makefile分析u-boot

  • u-boot启动的第一个文件：cpu/arm920T/start.s
  • 链接地址 board/你的板子的名字/u-boot.lds
    • 通过设置TEXT_BASE的内容我们可以将代码段在不同的内存地址上运行

      分为从nor flash启动还是从nand flash启动
      从nor flash直接复制
      从nand flash启动要进行初始化
      如何判断是从nor还是nand flash启动？
      对于norflash 必须通过一定的命令序列才能写数据。
      所以可以根据这点差别来分辨是从nor还是nand启动
      向地址0写入一个数据，然后读出来，如果没有改变的话就是nor flash。
